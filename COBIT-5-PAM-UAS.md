## üèõÔ∏è PRINSIP COBIT 5 (5 Prinsip Utama)

### 1. Meeting Stakeholder Needs
- Identifikasi stakeholder utama via **stakeholder matrix**
- **Goals Cascade**: Business Goals ‚Üí IT Goals ‚Üí Enabler Goals
- Prioritaskan berdasarkan **impact & value delivery**

### 2. Covering the Enterprise End-to-End
- Cakup **strategi ‚Üí operasi ‚Üí support**
- Hindari **TI silo** dengan governance terintegrasi

### 3. Applying a Single Integrated Framework
- COBIT 5 + ITIL + TOGAF + ISO 38500 + CMMI + PMBOK
- **Mapping Cross-Reference** untuk integrasi standar
- **Kurangi kompleksitas** governance

### 4. Enabling a Holistic Approach
- Kelola **interdependensi 7 enabler**
- Balance objectives: **effectiveness-efficiency-confidentiality**

### 5. Separating Governance from Management
- **GOVERNANCE (Board/Executive)**: EDM (Evaluate-Direct-Monitor)
- **MANAGEMENT (IT Management)**: APO-BAI-DSS-MEA (Plan-Build-Run-Monitor)
- **RACI chart** jelas untuk accountability

## üìä COBIT 5 PAM - Process Assessment Model (Detailed)

### Tujuan PAM
```
1. Capability Profile - Tentukan current capability organisasi
2. Strengths & Weaknesses - Identifikasi area improvement
3. Maturity Roadmap - Plan perbaikan berbasis bukti
4. ISO/IEC 15504 Compliant - Standar internasional SPICE
5. Benchmark - Compare dengan organisasi sejenis
```

### 5 Tahap Assessment PAM

#### Tahap 1: SCOPING
```
- Definisikan assessment scope
- Pilih proses yang relevan berdasarkan:
  * Goals cascade dari business strategy
  * Risk assessment
  * Regulatory requirements
- Create assessment team charter
```

#### Tahap 2: PLANNING
```
- Assemble assessment team
- Lead assessor harus CERTIFIED
- Define assessment method (document review, interview, observation)
- Create assessment timeline dan schedule
- Prepare data collection tools
```

#### Tahap 3: DATA COLLECTION
```
- Conduct interviews dengan process owners
- Document review (policies, procedures, records)
- Process observation (walk-through processes)
- Collect evidence untuk setiap base practice:
  * Management practices
  * Work products
  * Process attributes
- Verify evidence dengan multiple sources
```

#### Tahap 4: PROCESS RATING
```
- Rate capability level per proses (0-5)
- Assess process attributes (PA1.1 hingga PA5.2)
- Calculate Process Performance Indicators (PPI)
- Rating rule: ‚â•80% base practices = fully achieved
- Document rating justification dengan bukti
```

#### Tahap 5: REPORTING
```
- Create Process Capability Profile (Level 0-5 per proses)
- Identify strengths (good practices)
- Identify gaps dan weaknesses
- Create improvement recommendations
- Prioritize remediation actions
- Deliver formal assessment report
- Present findings to management
```

## üìà 6 LEVEL KAPABILITAS PAM + 9 PROCESS ATTRIBUTES

### Level 0: INCOMPLETE
```
- Proses TIDAK achieve purpose dasar
- NO Process Attributes
- Tidak ada bukti eksekusi proses
- Rating: Failed/Not Achieved
```

### Level 1: PERFORMED
```
- Proses deliver outcomes dasar
- Process Attributes: PA1.1 (Process Performance)
  * Process performance attribute mengukur extent proses deliver expected outcomes
  * Requires: ‚â•50% base practices achieved
- Contoh: Incident management handling terjadi tapi BELUM terstruktur
```

### Level 2: MANAGED
```
- Proses dikelola dengan planning & monitoring
- Process Attributes:
  * PA2.1 (Performance Management)
    - Proses direncanakan dan dimonitor
    - Work products diidentifikasi
    - Performance metrics didefinisikan
  * PA2.2 (Work Product Management)
    - Work products dikontrol
    - Requirements terpenuhi
    - Standards diterapkan
- Contoh: Incident management dengan SLA, KPI tracking
```

### Level 3: ESTABLISHED
```
- Proses didefinisikan sebagai standard organisasi
- Process Attributes:
  * PA3.1 (Process Definition)
    - Standard process terdefinisikan
    - Process tailoring rules
    - Process documentation lengkap
  * PA3.2 (Process Deployment)
    - Standard process deployed organisasi-wide
    - Procedure training diberikan
    - Monitoring compliance terhadap standard
- Contoh: Incident management dengan documented standard procedure di semua unit
```

### Level 4: PREDICTABLE
```
- Proses terkendali secara kuantitatif
- Process Attributes:
  * PA4.1 (Quantitative Analysis)
    - Quantitative objectives ditetapkan
    - Process performance dianalisis statistik
    - Control limits/thresholds ditentukan
  * PA4.2 (Process Control)
    - Process capability diukur dan dianalisis
    - Variation patterns diidentifikasi
    - Control actions untuk stabilitas
- Contoh: Incident MTTR (Mean Time To Resolution) terkontrol dengan target statistik
```

### Level 5: OPTIMIZING
```
- Proses terus ditingkatkan
- Process Attributes:
  * PA5.1 (Process Innovation)
    - Improvement opportunities diidentifikasi
    - Innovative approaches dievaluasi
    - New techniques diterapkan
  * PA5.2 (Process Optimization)
    - Process improvements diimplementasikan
    - Effectiveness improvements dimonitor
    - Lessons learned dikapturekan
- Contoh: Incident management dengan AI/ML untuk predictive analytics & auto-resolution
```

### Maturity Levels Progression
```
Level 0 (Incomplete)
  ‚Üì
Level 1 (Performed) - Proses berjalan
  ‚Üì
Level 2 (Managed) - Proses terkelola
  ‚Üì
Level 3 (Established) - Proses terstandar organisasi
  ‚Üì
Level 4 (Predictable) - Proses terkontrol kuantitatif
  ‚Üì
Level 5 (Optimizing) - Proses terus ditingkatkan
```

## ‚öñÔ∏è PERBANDINGAN COBIT 5 FRAMEWORK vs PAM

| Aspek | **COBIT 5 Framework** | **COBIT 5 PAM** |
|-------|----------------------|-----------------|
| **Definisi** | Governance framework holistik | Assessment model untuk kapabilitas proses |
| **Fokus Utama** | Design governance system | Measure capability level proses |
| **User/Pengguna** | Board, Executive, IT Management | Certified Assessor, Auditor, IT Governance Team |
| **Input** | Business requirements, stakeholder needs | Current process documentation, evidence |
| **Output** | Goals cascade, governance policies, RACI | Capability profile, maturity report, roadmap |
| **Komponen Kunci** | 5 Prinsip + 7 Enabler + 37 Proses | 6 Levels + 9 Process Attributes |
| **Standar** | ISACA Framework | ISO/IEC 15504:2012 (SPICE Standard) |
| **Proses** | 37 proses dengan practice definition | Assessment semua 37 proses |
| **Timeline** | Ongoing governance | Periodic assessment (annually/biannually) |
| **Tujuan** | Align TI-Business, Risk mitigation | Identify gaps, Prioritize improvements |
| **Deliverable** | Framework document, governance structure | Assessment report, capability scores |

### Pertanyaan 1: Prinsip COBIT 5
**Q**: Sebutkan 5 prinsip COBIT 5 dan jelaskan salah satunya!  
**A**: 
1. Meeting Stakeholder Needs - identifikasi kebutuhan stakeholder via goals cascade
2. Covering Enterprise End-to-End - cakup strategi hingga operasi
3. Single Integrated Framework - satukan ITIL, ISO, CMMI
4. Enabling a Holistic Approach - gunakan 7 enabler terintegrasi
5. Separating Governance from Management - beda EDM (gov) dan PBRM (mgmt)

### Pertanyaan 2: Domain COBIT 5
**Q**: Berapa total proses COBIT 5? Sebutkan 5 domain dan jumlah proses masing-masing!  
**A**: **Total 37 proses**
- EDM: 5 proses (Governance)
- APO: 13 proses (Align-Plan-Organize)
- BAI: 11 proses (Build-Acquire-Implement)
- DSS: 6 proses (Deliver-Service-Support)
- MEA: 5 proses (Monitor-Evaluate-Assess)

### Pertanyaan 3: EDM vs Management
**Q**: Jelaskan perbedaan EDM dan APO-BAI-DSS-MEA!  
**A**: 
- **EDM** = Governance level (Board/Executive), Evaluate-Direct-Monitor
- **APO-BAI-DSS-MEA** = Management level (IT Management), Plan-Build-Run-Monitor

### Pertanyaan 4: PAM Levels
**Q**: Jelaskan Level 3 Established dalam PAM dan berikan contoh!  
**A**: 
- **Level 3 Established**: PA3.1 (Process Definition) + PA3.2 (Process Deployment)
- Proses didefinisikan sebagai standard organisasi-wide
- Contoh: Incident management procedure didokumentasikan dan diterapkan di semua unit bisnis

### Pertanyaan 5: PAM Assessment
**Q**: Sebutkan 5 tahap assessment COBIT 5 PAM!  
**A**: 
1. Scoping - Pilih proses relevan berdasarkan prioritas
2. Planning - Bentuk assessment team, buat rencana
3. Data Collection - Interview, dokumen review, observasi
4. Process Rating - Rate capability levels per proses
5. Reporting - Capability profile, strengths/weaknesses, improvement plan

### Pertanyaan 6: Process Attributes
**Q**: Apa saja 9 Process Attributes dalam PAM? Jelaskan PA2.1 dan PA2.2!  
**A**: 
9 PA: PA1.1, PA2.1, PA2.2, PA3.1, PA3.2, PA4.1, PA4.2, PA5.1, PA5.2

- **PA2.1 (Performance Management)**: Proses direncanakan & dimonitor, metrics ditetapkan
- **PA2.2 (Work Product Management)**: Work products dikontrol, requirements terpenuhi, standards diterapkan

### Pertanyaan 7: Goals Cascade
**Q**: Apa itu Goals Cascade dalam COBIT 5? Berikan contoh!  
**A**: 
Terjemahan hierarchical dari Business Goals ‚Üí IT-Related Goals ‚Üí Enabler Goals
- Business Goal: "Increase market share by 20%"
- IT Goal: "Improve service availability to 99.9%"
- Enabler Goal: "Establish robust infrastructure & disaster recovery"

### Pertanyaan 8: 7 Enabler
**Q**: Sebutkan 7 enabler COBIT 5 dan jelaskan enabler no. 2!  
**A**: 
1. Principles, Policies, Frameworks
2. **Processes** ‚Üê 37 proses dengan purpose, practices, RACI
3. Organizational Structures
4. Culture, Ethics, Behavior
5. Information
6. Services, Infrastructure, Applications
7. People, Skills, Competencies

### Pertanyaan 9: APO12 & DSS05
**Q**: Jelaskan proses APO12 (Manage Risk) dan DSS05 (Manage Security)!  
**A**: 
- **APO12**: Risk identification, assessment, response; risk management framework
- **DSS05**: Access management, authentication, encryption, intrusion detection, incident response

### Pertanyaan 10: Maturity Assessment Output
**Q**: Apa output dari COBIT 5 PAM assessment?  
**A**: 
- Process Capability Profile (Level 0-5 per proses)
- Strengths dan weaknesses identification
- Improvement recommendations
- Improvement roadmap dengan prioritas
- Formal assessment report
